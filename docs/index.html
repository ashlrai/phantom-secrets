How Phantom works

Two commands. No code changes. Your existing workflow stays exactly the same.

1

Protect your secrets

Run phantom init in your project. Real secrets are moved to your OS keychain. Your .env is rewritten with worthless phm_ tokens. Auto-detects OpenAI, Anthropic, Stripe, and 10+ services.

2

Code with AI safely

Run phantom exec -- claude. A local HTTP proxy starts. The AI reads your .env and only sees phantom tokens. Session-scoped — fresh tokens every time.

3

Secrets injected at network layer

When your code makes an API call, the proxy intercepts it, swaps the phantom token for the real credential, and forwards over TLS. The AI never saw the real key.

4

Deploy anywhere

Run phantom sync --platform vercel to push secrets to your deployment platform. Pull from Vercel or Railway to onboard new machines. One source of truth.

Built for the AI coding era

Every feature exists to keep your secrets safe while letting AI do its job.

🔐

Encrypted Vault

ChaCha20-Poly1305 encryption with Argon2id key derivation. OS keychain on macOS/Linux, encrypted file for CI.

🔄

Session Tokens

Fresh phantom tokens every session. If one leaks from AI logs, it's already worthless. The real secret was never exposed.

🤖

MCP Server

Native Claude Code integration. AI manages secrets through tools — never sees values. phantom setup configures it.

🛡

Pre-commit Hook

phantom check blocks commits containing real secrets. Catches hardcoded API keys in staged files.

☁

Platform Sync

Push to Vercel and Railway. Pull to onboard new machines. No more copying secrets through Slack.

🔍

Smart Detection

Auto-detects 13+ services from key names. Skips non-secrets like NODE_ENV and PORT.

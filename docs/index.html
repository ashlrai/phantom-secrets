Two commands. No code changes.

Your existing workflow stays exactly the same. Phantom works underneath.

1

Protect your secrets

Run phantom init. Real secrets move to your OS keychain (encrypted with ChaCha20). Your .env is rewritten with phm_ tokens that are cryptographically worthless.

2

Code with AI safely

Run phantom exec -- claude. A local proxy starts with fresh session tokens. The AI reads your .env and sees nothing useful. If tokens leak, they're already dead.

3

Proxy injects real keys

When your code makes an API call, the proxy intercepts it, swaps the phantom token for the real credential, and forwards over TLS. Your code works perfectly. The AI never knew.

4

Deploy and sync

Run phantom sync --platform vercel to push secrets to your deployment platform. phantom pull --from vercel to onboard a new machine. One source of truth.

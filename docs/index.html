How Phantom works

Two commands. No code changes. Your existing workflow stays the same.

1

Protect your secrets

Run phantom init. Real secrets move to your OS keychain (encrypted). Your .env is rewritten with phm_ tokens. Auto-detects OpenAI, Stripe, and 13+ services.

2

Code with AI safely

Run phantom exec -- claude. A local proxy starts. The AI reads your .env and only sees phantom tokens. Fresh session tokens every time — if one leaks, it's already dead.

3

Proxy injects real credentials

When your code hits an API, the request goes through the local proxy. It swaps the phantom token for your real key and forwards over TLS. Your code works. The AI never knew.

4

Deploy and sync

Run phantom sync --platform vercel to push real secrets to your deployment platform. Pull from Vercel or Railway to onboard a new machine. One source of truth.

Built for the AI coding era

Every feature exists to keep your secrets safe while AI does its job.

Encrypted vault

ChaCha20-Poly1305 encryption. Argon2id key derivation. OS keychain on macOS/Linux. Encrypted file fallback for CI.

Session tokens

Fresh phantom tokens every session. If one leaks from AI logs, it's already worthless. Real secrets were never exposed.

MCP server

Native Claude Code integration. AI manages secrets through MCP tools — never sees real values. One command to configure.

Pre-commit hook

phantom check blocks commits containing real secrets. Catches hardcoded API keys in staged files.

Platform sync

Push to Vercel and Railway. Pull to onboard new machines. No more pasting secrets into dashboards.

Smart detection

Auto-detects 13+ API services from key names. Leaves NODE_ENV and PORT untouched.

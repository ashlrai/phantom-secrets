How It Works

Four steps to zero leaked secrets

Phantom replaces your real credentials with worthless tokens. A local proxy injects the real secrets at the network layer.

🔒
01

Initialize

Run phantom init to scan your .env, encrypt secrets into your OS keychain, and rewrite the file with phantom tokens.

▶
02

Execute

Run phantom exec -- claude to launch your AI tool. It only ever sees phm_ tokens in memory.

🔃
03

Proxy Injects

A local HTTP proxy intercepts outbound API calls, swaps phantom tokens for real credentials, and forwards over TLS.

🚀
04

Deploy

Run phantom sync to push secrets to Vercel, Railway, or any platform. One source of truth.

Features

Everything you need to protect secrets

Built for developers who ship with AI. Zero configuration for common setups.

🔐

Encrypted Vault

Secrets stored in your OS keychain (macOS Keychain, Windows Credential Manager, Linux Secret Service). Protected by your login.

🕐

Session Tokens

Phantom tokens rotate per session. Even if a token leaks, it expires when the session ends. Zero persistent exposure.

🔌

MCP Server

Native Claude Code integration via Model Context Protocol. Manage secrets without values entering the AI context window.

🚫

Pre-commit Hook

Blocks commits containing unprotected secrets. Catches hardcoded API keys, connection strings, and tokens in your code.

☁

Platform Sync

Push secrets to Vercel and Railway. Pull to onboard new machines. Stop copy-pasting env vars between dashboards.

🔍

Smart Detection

Automatically distinguishes secrets from config. Leaves NODE_ENV, PORT, and DEBUG untouched. Zero false positives.

👁

Open Source

MIT licensed. Written in Rust. Fully auditable, forkable, and free forever for individuals and teams.

📦

npm Install

Run npx phantom-secrets for instant setup. No Rust toolchain required. Works everywhere Node runs.

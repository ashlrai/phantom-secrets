The Problem

AI coding tools read your .env file. Your secrets enter the LLM context window. They can leak via prompt injection, session logs, or training data.

Without Phantom

.env:
OPENAI_API_KEY=sk-real-secret-key

With Phantom

.env:
OPENAI_API_KEY=phm_a7f3b9e2...

AI sees: phm_a7f3b9e2... (worthless)
Proxy injects: real key at network layer

How It Works

1

Initialize

Run phantom init in your project. Real secrets are stored in your OS keychain. Your .env is rewritten with worthless phantom tokens.

2

Code with AI

Run phantom exec -- claude. The AI reads your .env and only sees phm_ tokens. If they leak, nothing happens.

3

Proxy injects secrets

A local HTTP proxy intercepts API calls, replaces phantom tokens with real credentials, and forwards over TLS. The AI never sees the real secret.

Features

MCP Server

Native Claude Code integration. Manage secrets without values entering the AI context.

Pre-commit Hook

Block commits containing unprotected secrets. Catches hardcoded API keys in code.

OS Keychain

Secrets stored in macOS Keychain or Linux Secret Service. Protected by your login password.

Smart Detection

Automatically identifies secrets vs config. Leaves NODE_ENV and PORT untouched.
